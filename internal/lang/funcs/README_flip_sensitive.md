# Testing the `flip_sensitive` Function

This guide explains how to test the `flip_sensitive` function using the provided Terraform configuration file.

## Prerequisites

- Pull the code from the branch from which this pr is opened and run `make build` inside the OpenTofu repo to produce a binary.
- Then copy the tofu binary generated by the `make build` command to your working directory.
- In the same working directory, create a file named `test_flip_sensitive.tf` with the following content:

```hcl
# This file demonstrates how to use the flip_sensitive function in OpenTofu

# Define a sensitive variable
variable "sensitive_password" {
   type      = string
   sensitive = true
   default   = "super-secret-password"
}

# Define a non-sensitive variable
variable "non_sensitive_data" {
   type    = string
   default = "public-information"
}

# Use flip_sensitive to make sensitive data non-sensitive
locals {
   # This will make the sensitive password non-sensitive
   exposed_password = flip_sensitive(var.sensitive_password)

   # This will make the non-sensitive data sensitive
   protected_data = flip_sensitive(var.non_sensitive_data)
}

# Output to demonstrate the results
output "original_sensitive_password" {
   value       = var.sensitive_password
   sensitive   = true
}

output "flipped_password" {
   value       = local.exposed_password
   # No need to mark as sensitive since flip_sensitive removed the sensitivity
}

output "original_non_sensitive_data" {
   value = var.non_sensitive_data
}

output "flipped_data" {
   value     = local.protected_data
   sensitive = true  # This is redundant as the value is already sensitive, but added for clarity
}

# You can also use flip_sensitive with complex types
locals {
   sensitive_map = {
      username = "admin"
      password = "secret"
   }

   # Make the entire map sensitive
   protected_map = sensitive(local.sensitive_map)

   # Flip the sensitivity (make it non-sensitive)
   exposed_map = flip_sensitive(local.protected_map)
}

output "protected_map" {
   value     = local.protected_map
   sensitive = true
}

output "exposed_map" {
   value = local.exposed_map
}
```

## Running the Test

1. Initialize your OpenTofu working directory:
   ```
   tofu init
   ```

2. Apply the configuration:
   ```
   tofu apply
   ```

3. When prompted, type `yes` to apply the changes.

## Expected Output

When you run `tofu apply`, you should see output similar to the following:

```
Apply complete! Resources: 0 added, 0 changed, 0 destroyed.

Outputs:

exposed_map = {
  "password" = "secret"
  "username" = "admin"
}
flipped_password = "super-secret-password"
original_non_sensitive_data = "public-information"
original_sensitive_password = (sensitive value)
protected_map = (sensitive value)
flipped_data = (sensitive value)
```

## Understanding the Results

- `original_sensitive_password`: Shows as "(sensitive value)" because it's marked as sensitive
- `flipped_password`: Shows the actual password value because `flip_sensitive` removed the sensitivity mark
- `original_non_sensitive_data`: Shows the actual value because it's not sensitive
- `flipped_data`: Shows as "(sensitive value)" because `flip_sensitive` made it sensitive
- `protected_map`: Shows as "(sensitive value)" because it was marked as sensitive using the `sensitive()` function
- `exposed_map`: Shows the actual map values because `flip_sensitive` removed the sensitivity mark

## Verifying with Output Command

You can also use the `tofu output` command to verify the sensitivity status:

```
# This will show "(sensitive value)"
tofu output original_sensitive_password

# This will show the actual value
tofu output flipped_password

# This will show the actual value
tofu output original_non_sensitive_data

# This will show "(sensitive value)"
tofu output flipped_data

# This will show "(sensitive value)"
tofu output protected_map

# This will show the actual map values
tofu output exposed_map
```

To see the actual values of sensitive outputs, use the `-json` flag:

```
tofu output -json
```

This will show all values, including sensitive ones, in JSON format.

## How It Works

The `flip_sensitive` function:
1. Takes a single argument of any type
2. If the input is sensitive, it returns the same value without the sensitive mark
3. If the input is not sensitive, it returns the same value with a sensitive mark

This is useful for:
- Exposing sensitive values in controlled environments
- Making non-sensitive values sensitive for additional protection
- Controlling the sensitivity of values in complex data structures
